# Claude Code Skills — Engineer Online

本项目为 Engineer Online 提供 7 个 Claude Code skill，覆盖全部开发角色和评审场景。开发人员只需一句简单指令即可触发完整的 AI coding 工作流。

## Skill 列表

| Skill | 角色 | 适用模块 | 触发示例 |
|-------|------|----------|----------|
| `flutter-dev` | Flutter 移动端 | 5.2/5.3/5.5/5.6 | `/flutter-dev 完成任务 QLIST-04` |
| `h5-dev` | H5 前端 | 5.4 | `/h5-dev 完成任务 QDETAIL-05` |
| `admin-dev` | 管理后台前端 | 5.1/5.7/5.8/5.9 | `/admin-dev 完成任务 GROUP-05` |
| `backend-dev` | Java 后端 | 全部 | `/backend-dev 完成任务 QPOST-02` |
| `db-design` | 数据库设计 | 全部 | `/db-design 完成任务 GROUP-01` |
| `test-qa` | 测试 | 全部 | `/test-qa 完成任务 TEST-02` |
| `spec-writer` | Spec 文档编写 | 新模块/新功能 | `/spec-writer 为圈子管理编写 spec` |
| `spec-review` | 评审 HTML 生成 | 评审会议 | `/spec-review 生成本次评审 HTML` |

## 使用方式

### 方式 1：按任务编号（推荐）

```
/flutter-dev 完成任务 QLIST-04
```

### 执行流程（人在回路）

每个 skill 遵循 **"先确认方案，再生成代码"** 的原则：

```
用户输入 → AI 解析 → 输出实施方案 → 等待用户确认 → 加载完整上下文 → 生成代码 → 自检清单
```

#### 步骤 1：AI 输出实施方案

AI 会读取 `spec-index.yaml` 和对应 PRD 的核心章节，输出方案表格：

```markdown
## 实施方案

| 项目 | 内容 |
|------|------|
| **任务编号** | QLIST-04 |
| **目标模块** | 5.2 圈子首页 |
| **功能描述** | 实现首页页面（品牌区+统计+Tab+问题列表+Ask按钮） |
| **输出文件** | lib/features/home/view/home_page.dart 等 |
| **需加载文档** | 05.02-圈子首页.md、技术架构.md 3.1节、UI设计.md |
| **预估范围** | 页面+Controller+Repository+Widgets+路由 |

### 关键业务规则（预览）
- BR-5.2-01: 仅显示审核通过且公开的问题
- BR-5.2-06: cursor 分页，每次 20 条
- ...

---
**请确认此方案后继续。如有调整请直接说明。**
```

#### 步骤 2：用户确认

- **确认**：回复"确认"/"OK"/"没问题" → AI 进入代码生成
- **调整**：回复"不需要 Ask 按钮"、"先只做 UI 部分" → AI 更新方案后再次等待确认

#### 步骤 3：AI 生成代码 + 自检清单

确认后 AI 加载完整上下文，生成代码并输出自检清单。

### 方式 2：按模块 ID

```
/flutter-dev 为模块 5.2 生成首页代码
```

### 方式 3：自由描述

```
/backend-dev 帮我实现问题创建的 Service 层，包含敏感词校验和审核状态机
```

## 方案确认的意义

| 场景 | 作用 |
|------|------|
| **确认理解正确** | 确保 AI 没有误解任务（如"我以为你说的是 5.3，结果是 5.2"） |
| **控制范围** | 避免 AI 生成过多代码（如"先做 UI，API 联调后面再说"） |
| **提前发现问题** | 方案阶段就能发现需求模糊、文档缺失等问题 |
| **节省 Token** | 方案确认前只加载最小上下文，避免加载大量文档后发现方向错误 |

## Skill 设计原则

1. **零配置**：skill 内置所有文档加载逻辑，开发人员不需要知道文档路径
2. **智能解析**：自动从任务编号推断模块、技术栈、输入文档
3. **人在回路**：先输出方案供确认，避免"生成一堆代码后发现方向错了"
4. **上下文预算管理**：方案阶段只加载最小上下文，确认后才加载完整文档
5. **自检清单**：每个 skill 输出末尾强制要求列出 BR/EX/AC 覆盖情况

## 与 Kiro Skills 的关系

| | Kiro Skills (`.kiro/skills/`) | Claude Code Skills (`.claude/skills/`) |
|---|---|---|
| **调用方式** | `#flutter-dev` | `/flutter-dev` |
| **上下文加载** | 需手动指定文档 | 自动按内置顺序加载 |
| **任务解析** | 需手动指定模块 | 自动从任务编号解析 |
| **方案确认** | 无 | 强制先输出方案，等待确认 |
| **自检清单** | 静态模板 | 动态生成（基于实际读取的文档） |

两套 skill 内容同源，Claude Code 版本增加了**自动化**和**方案确认**能力。
