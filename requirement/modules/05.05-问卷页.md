## §05.05.3 交互逻辑

### 主流程：打开问卷页 → 提交 → 完成回流

```mermaid
flowchart TD
    A[从浮标进入问卷页] --> B[WebView 加载 CSI H5 URL<br/>已含 ?oneId=xxx&campaignCode=xxx]
    B --> C{加载成功?}
    C -->|否| D[显示重试页面]
    C -->|是| E[用户填写问卷]
    E --> F[点击 H5 Submit]
    F --> G[H5 提交数据到 CSI 成功]
    G --> H[H5 调用 JSBridge<br/>npsSurveyCompleted]
    H --> I[App 立即隐藏浮标<br/>并返回车控页]
    I --> J[App 后台异步调用<br/>POST /complete]
    J --> K{调用成功?}
    K -->|是| L[标记 completed = true<br/>流程结束]
    K -->|否| M[本地保存 pending 记录<br/>{campaignCode, oneId, createdAt}]
    M --> N[下次 App 启动时<br/>检查 pending 队列]
    N --> O{createdAt ≤ 1 小时?}
    O -->|是| P[重试 POST /complete]
    P --> Q{成功?}
    Q -->|是| R[删除 pending 记录]
    Q -->|否| S[保留 pending 记录<br/>等下次启动再重试]
    O -->|否| T[最后一次重试]
    T --> U{成功?}
    U -->|是| V[删除 pending 记录]
    U -->|否| W[标记失效 / 删除<br/>不再重试]
```

### JSBridge 协议

**H5 → App**：

```typescript
window.flutterBridge.postMessage(JSON.stringify({
  method: 'npsSurveyCompleted',
  params: {
    campaignCode: 'NPS20260001',
    oneId: 'one192929293939'
  }
}));
```

**App 行为**：
1. 接收 `campaignCode` + `oneId`。
2. 立即隐藏浮标并返回车控页（用户视角：已完成）。
3. 后台异步调用 `POST /api/v1/campaigns/user/surveys/complete`。
4. 调用成功则标记完成；调用失败则本地保存一条 pending 记录（含 `campaignCode`、`oneId`、`createdAt` 时间戳），用户无感知。
5. 每次 App 启动时，遍历 pending 队列逐一重试 `POST /complete`：
   - 若记录创建时间距今 ≤ 1 小时：重试；成功则删除记录，失败则保留等下次启动
   - 若记录创建时间距今 > 1 小时：最后一次重试；成功则删除，失败则标记失效/删除不再重试
6. 收到 4105（记录未找到）直接标记失效不重试；收到 4104（已完成）视为成功删除记录。

## §05.05.4 业务规则

- **BR-05.05-01** 问卷页由 CSI 系统提供的 H5 URL 渲染，App 仅做 WebView 容器；后端问卷查询接口返回的 URL 已包含 `?oneId=xxx&campaignCode=xxx` 参数，App 直接使用即可，无需二次拼接，CSI H5 据此识别用户身份。（→ AC-05.05-01）
- **BR-05.05-02** 提交按钮位于 H5 内部，由 CSI H5 实现；H5 将问卷数据提交至 CSI 成功后，须调用 App 原生 JSBridge 方法 `npsSurveyCompleted` 并传入 `campaignCode`、`oneId` 两个参数。（→ AC-05.05-02）
- **BR-05.05-03** App 收到 JSBridge 回调后立即隐藏浮标并返回车控页（用户侧完成），同时后台异步调用服务端完成接口 `POST /api/v1/campaigns/user/surveys/complete`；成功后将投放记录标记为已完成。（→ AC-05.05-03）
- **BR-05.05-04** 完成接口需幂等处理；已完成的记录再次调用时返回成功，不重复更新。（→ AC-05.05-04）
- **BR-05.05-05** WebView 加载 H5 失败时显示重试页面，用户可点击重试或返回车控页。（→ AC-05.05-05）
- **BR-05.05-06** H5 与 App 的通信须兼容 WebView 安全区、返回手势及主题同步规范（参考 `standard/技术架构.md` 第 8 章 JSBridge 模式）。（→ AC-05.05-06）
- **BR-05.05-07** 后台完成接口调用失败时，本地保存一条 pending 记录（含 `campaignCode`、`oneId`、`createdAt` 时间戳）；每次 App 启动时遍历 pending 队列逐一重试 `POST /complete`。记录创建 1 小时内：失败则保留等下次启动再重试；超过 1 小时：最后一次重试仍失败则标记失效/删除，不再重试。服务端通过 `oneId + campaignCode` 定位投放记录并标记完成。收到 4105（记录未找到）直接失效不重试；收到 4104（已完成）视为成功。重试期间不影响用户正常使用 App。（→ AC-05.05-07, AC-05.05-08）
- **BR-05.05-08** 投放计划停用不影响进行中的问卷提交：用户在计划停用前已打开问卷页的，完成上报接口照常接收并按原流程处理（幂等、重试规则不变）；停用的影响范围仅限定时下发与问卷查询接口返回（见 BR-05.01-05）。（→ AC-05.05-11）

## §05.05.5 异常处理

| 编号          | 场景                 | 触发条件                       | 系统行为              | 用户提示             | 恢复方式                     |
| ----------- | ------------------ | -------------------------- | ----------------- | ---------------- | ------------------------ |
| EX-05.05-01 | WebView 加载 H5 失败   | 网络异常或 URL 无效               | 显示重试页面            | 提示”加载失败，点击重试”    | 点击重试或返回                  |
| EX-05.05-02 | JSBridge 调用异常      | H5 调用原生方法失败或参数缺失           | App 记录日志          | 无（用户侧已提交 CSI 成功） | 无                        |
| EX-05.05-03 | 完成接口首次失败           | 服务端异常或网络问题                 | 本地保存 pending 记录，用户无感知    | 无                | 下次 App 启动时自动重试          |
| EX-05.05-04 | 重复完成上报             | 用户已完成问卷后再次调用               | 幂等返回成功            | 无                | 无                        |
| EX-05.05-05 | H5 未实现 JSBridge 回调 | CSI H5 未接入                 | 提交后无响应            | 无（由 CSI 侧修复）     | 由 CSI 侧修复 H5             |

| EX-05.05-08 | 重复完成上报             | 投放记录已完成                    | 幂等返回成功（code 4104） | 无                | 无需处理                     |

## §05.05.10 验收标准

### 正常流程

- **AC-05.05-01**: **Given** 用户从浮标进入问卷页，**When** WebView 加载 CSI H5 URL（URL 已由后端拼接 `?oneId=xxx&campaignCode=xxx` 参数），**Then** 页面正确渲染问卷内容且 CSI H5 可读取 oneId 和 campaignCode 识别用户。（← BR-05.05-01）
- **AC-05.05-02**: **Given** 用户填写完问卷且 H5 已提交到 CSI，**When** 点击 H5 内 Submit 按钮，**Then** H5 调用 JSBridge `npsSurveyCompleted` 并传入 `campaignCode` + `oneId`。（← BR-05.05-02）
- **AC-05.05-03**: **Given** App 收到 JSBridge 完成回调，**When** 处理回调，**Then** 立即隐藏浮标并返回车控页，同时后台异步调用 `POST /api/v1/campaigns/user/surveys/complete`。（← BR-05.05-03）
- **AC-05.05-04**: **Given** 用户已完成问卷，**When** H5 再次触发完成回调，**Then** 服务端幂等返回成功，不重复更新。（← BR-05.05-04）

### 异常流程

- **AC-05.05-05**: **Given** WebView 加载 H5 失败，**When** 页面显示重试页，**Then** 用户可点击重试或返回车控页。（← EX-05.05-01）
- **AC-05.05-06**: **Given** H5 Submit 后 JSBridge 调用失败，**When** App 未收到回调，**Then** 浮标不隐藏，用户可重新进入问卷页提交。（← EX-05.05-02）
- **AC-05.05-07**: **Given** 后台完成接口首次失败后本地保存了 pending 记录，**When** 下次 App 启动时重试成功，**Then** 删除 pending 记录，服务端将投放记录标记为 completed = true。（← BR-05.05-07）
- **AC-05.05-08**: **Given** pending 记录创建超过 1 小时，**When** App 启动后最后一次重试仍失败，**Then** 标记失效/删除 pending 记录，不再重试，本地记录日志，服务端保持未完成状态。（← BR-05.05-07, EX-05.05-06）

### 边界测试

- **AC-05.05-09**: **Given** 用户在弱网环境下提交问卷，**When** 首次完成接口超时，**Then** 本地保存 pending 记录，用户侧已显示成功（浮标已隐藏），下次 App 启动时自动重试。（← EX-05.05-03）
- **AC-05.05-10**: **Given** H5 未接入 JSBridge，**When** 用户点击 Submit，**Then** App 检测到无回调，浮标不隐藏。（← EX-05.05-05）
- **AC-05.05-11**: **Given** 用户已打开问卷页，随后投放计划被停用，**When** 用户提交问卷且 App 后台调用完成接口，**Then** 服务端照常接收并将投放记录标记为已完成。（← BR-05.05-08）

### §05.05.11.1 完成问卷（App 客户端 → App 服务端）

> 本接口由 App 在后台异步调用，不影响用户正常使用。服务端通过 `oneId + campaignCode` 定位投放记录并标记完成。失败时 App 本地保存 pending 记录，每次 App 启动时重试；创建超过 1 小时的记录最后一次重试仍失败则失效不再重试。

**请求**：

```http
POST /api/v1/campaigns/user/surveys/complete
Authorization: Bearer {jwt}
Content-Type: application/json

{
  "campaignCode": "NPS20260001",
  "oneId": "one192929293939"
}
```

**响应（成功）**：

```json
{
  "code": 0,
  "message": "success",
  "data": null,
  "traceId": "trace-nps-013",
  "timestamp": 1751328000000
}
```

**响应（错误 - 已完成）**：

```json
{
  "code": 4104,
  "message": "Survey already completed",
  "data": {
    "campaignCode": "NPS20260001",
    "oneId": "one192929293939"
  },
  "traceId": "trace-nps-014",
  "timestamp": 1751328000000
}
```

**响应（错误 - 未找到匹配记录）**：

```json
{
  "code": 4105,
  "message": "Delivery record not found",
  "data": {
    "campaignCode": "NPS20260001",
    "oneId": "one192929293939"
  },
  "traceId": "trace-nps-015",
  "timestamp": 1751328000000
}
```

**响应（错误 - 未登录）**：

```json
{
  "code": 2001,
  "message": "Unauthorized",
  "data": null,
  "traceId": "trace-nps-016",
  "timestamp": 1751328000000
}
```

### §05.05.11.2 JSBridge 调用示例

**H5 侧**：

```typescript
async function submitSurvey() {
  if (!window.flutterBridge) {
    // 非 App 环境兜底处理
    return;
  }
  try {
    window.flutterBridge.postMessage(JSON.stringify({
      method: 'npsSurveyCompleted',
      params: {
        campaignCode: 'NPS20260001',
        oneId: 'one192929293939'
      }
    }));
  } catch (e) {
    console.error('Survey completion report failed', e);
  }
}
```

**App 侧**：

```dart
// 注册 JSBridge 通道
_webViewController.addJavaScriptChannel(
  'flutterBridge',
  onMessageReceived: (JavaScriptMessage message) {
    final payload = jsonDecode(message.message);
    if (payload['method'] == 'npsSurveyCompleted') {
      final params = payload['params'];
      final campaignCode = params['campaignCode'];
      final oneId = params['oneId'];
      // 立即隐藏浮标，后台异步完成状态同步（含重试）
      _hideFloatingEntry();
      _completeSurveyInBackground(campaignCode: campaignCode, oneId: oneId);
    }
  },
);
```
