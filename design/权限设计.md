# 权限设计

> **文档版本**: 1.0.0  
> **更新日期**: 2026-07-09  
> **适用范围**: NPS 问卷投放模块  
> **关联文档**: `requirement/00-需求总览.md`, `requirement/modules/01.01.1~01.01.5`

---

## 1. 命名规范

Permission Code 格式：

```
{资源}:{动作}:{范围}
```

- **资源**：英文小写名词，如 `campaign`、`campaign_delivery`
- **动作**：英文小写动词，如 `view`、`create`、`update`、`delete`、`enable`、`import`、`export`、`push`
- **范围**：
  - `any`：任意对象
  - `own`：仅自己创建的对象

---

## 2. 角色定义

| 角色 | 说明 | 默认权限 |
|------|------|----------|
| 运营人员（OPERATOR） | 系统管理员，拥有全部权限 | 全部 Permission Code |
| 运营人员（OPERATOR） | 负责投放计划与投放数据日常运营 | `campaign:view:any`、`campaign:create:any`、`campaign:update:any`、`campaign:enable:any`、`campaign_delivery:view:any`、`campaign_delivery:import:any`、`campaign_delivery:export:any`、`campaign_delivery:push:any` |
| 车主用户（CAR_OWNER） | GAC APP 已绑车用户 | 通过车主身份隐式拥有查询/完成自己问卷的权限，不绑定具体 Permission Code |
| CSI 系统 | 外部系统 | 通过服务间密钥认证，不绑定 Permission Code |

## 4. 权限校验方式

### 4.1 后端校验

使用 Sa-Token 注解：

```java
@SaCheckPermission("campaign:create:any")
@PostMapping("/api/v1/admin/campaigns")
public Result<Long> create(@RequestBody @Valid CampaignCreateDTO dto) {
    return Result.ok(campaignService.create(dto));
}
```

### 4.2 前端校验

管理后台使用 `v-permission` 指令控制按钮渲染：

```vue
<a-button v-permission="['campaign:create:any']">创建投放计划</a-button>
<a-button v-permission="['campaign:delete:any']">删除</a-button>
```

无权限时元素不渲染（而非禁用）。

### 4.3 车主端权限

车主端接口通过 JWT 中的 `one_id` 校验，只能操作当前登录用户自己的投放记录：

```java
@GetMapping("/api/v1/campaigns/user/surveys")
public Result<List<CampaignSurveyVO>> list(@RequestHeader("X-User-Id") String oneId) {
    return Result.ok(deliveryService.listByUser(oneId));
}
```

## 6. 审计日志

所有运营人员对投放计划与投放数据的写操作均记录审计日志：

| 字段 | 说明 |
|------|------|
| 操作人 | 管理员/运营人员 ID |
| 操作类型 | CREATE / UPDATE / DELETE / ENABLE / DISABLE / IMPORT / EXPORT / PUSH |
| 资源类型 | CAMPAIGN / CAMPAIGN_DELIVERY |
| 资源 ID | 计划 ID 或投放记录 ID |
| 变更前后值 | JSON 格式 |
| 操作时间 | TIMESTAMP |
| IP 地址 | 操作来源 IP |

审计日志写入 `t_community_audit_log`（或本项目独立审计表），保留 1 年。
